# Query NebulaGraph metrics

You can query performance-related metrics in NebulaGraph, including configuring metric structure and querying metrics via HTTP.

## 1. Metrics Structure

Each metric in NebulaGraph consists of three fields: name, type, and time range.  
The fields are separated by periods, for example: `num_queries.sum.600`.

Different NebulaGraph services (Graph, Storage, or Meta) support different metrics.

| Field        | Example         | Description |
|--------------|-----------------|-------------|
| Metric name  | `num_queries`   | Indicates the function of the metric. |
| Metric type  | `sum`           | Indicates how the metrics are collected. Supported types: `sum`, `avg`, `rate`, and quantiles such as `p75`, `p95`, `p99`, `p999`. |
| Time range   | `600`           | Time range in seconds for metric collection. Supported values: `5`, `60`, `600`, `3600` representing the last 5 seconds, 1 minute, 10 minutes, and 1 hour. |


## 2. Querying Metrics via HTTP

You can query NebulaGraph metrics over HTTP.

---

### 2.1 Syntax and Parameters

```bash
curl -G "http://<host>:<port>/stats?stats=<metric_name_list>[&format=json]"
```

|Parameter|Description|
|-|-|
|`host`|The IP (or hostname) of the server. You can find it in the configuration file in the installation directory.|
|`port`|The HTTP port of the server. You can find it in the configuration file in the installation directory. The default ports are 19559 (Meta), 19669 (Graph), and 19779 (Storage).|
|`metric_name_list`|The metrics names. Multiple metrics are separated by commas (,).|
|`&format=json`|Optional. Returns the result in the JSON format.|

!!! note

    1. If NebulaGraph is deployed with [Docker Compose](../2.quick-start/1.quick-start-workflow.md), run `docker-compose ps` to check the ports that are mapped from the service ports inside of the container and then query through them.
    2. The HTTP port and host information depend on your service configuration.


### 2.2 Query Examples
#### Query a Single Metric
Query the number of queries in the last 10 minutes in the Graph service.

```bash
curl -G "http://192.168.8.40:19669/stats?stats=num_queries.sum.600"
```
Example output:
num_queries.sum.600=400

#### Query Multiple Metrics
Supports querying both metrics together in Meta service.
- The average heartbeat latency in the last 1 minute.
- The P99 heartbeat latency (slowest 1%) in the last 10 minutes.

```bash
curl -G "http://192.168.8.40:19559/stats?stats=heartbeat_latency_us.avg.60,heartbeat_latency_us.p99.600"
```
Example output: 
heartbeat_latency_us.avg.60=281
heartbeat_latency_us.p99.600=985

#### Return JSON Result

Query the number of new vertices in the Storage service in the last 10 minutes, returning the result in JSON.

```bash
curl -G "http://192.168.8.40:19779/stats?stats=num_add_vertices.sum.600&format=json"
```
Example output: [{"value":1,"name":"num_add_vertices.sum.600"}]

## 3. Space-Level Metrics
The Graph service supports space-level metrics, which record information for each graph space separately.
### 3.1 Query Format
Space-level metrics are returned only when you query all metrics.  
For example, run `curl -G "http://192.168.8.40:19559/stats"` to show all metrics.

### 3.2 Enable Query
1. Add to Graph service configuration:
   Set the value of `enable_space_level_metrics` to `true` in the Graph service configuration file before starting NebulaGraph.
2. Restart Graph service:
 The returned result contains the graph space name in the form of '{space=space_name}', such as `num_active_queries{space=basketballplayer}.sum.5=0`.

!!! note

    For details about how to modify the configuration, see [Configuration Management](../5.configurations-and-logs/1.configurations/1.configurations.md).
